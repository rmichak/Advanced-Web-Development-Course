1

Create a Supabase Account and Project

Supabase gives you a full Postgres database in the cloud for free. You'll set up an account and create a project to store your todo data.

Go to supabase.com in your browser and click "Start your project." Sign up with your GitHub account — it's the fastest option and connects nicely later.

Once you're logged in:

  1. Click "New Project"
  2. Name it react-todo-db
  3. Pick the closest region — East US works for most of us
  4. Set a database password (you won't need it directly, but remember it)
  5. Click "Create new project"

It takes about 2 minutes for Supabase to spin up your database. You'll see a loading screen — just wait for it to finish.

Free tier

Supabase's free tier includes 500 MB of database storage, 50,000 monthly active users, and 5 GB of bandwidth. More than enough for this assignment and any personal projects you want to experiment with.

2

Create the Todos Table

Your database needs a table to store todos. You'll create one using SQL in the Supabase dashboard.

In the Supabase dashboard, click SQL Editor in the left sidebar. This gives you a place to run SQL commands directly against your Postgres database. Paste the following SQL and click Run:

CREATE TABLE todos (
  id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  text text NOT NULL,
  created_at timestamptz DEFAULT now()
);

ALTER TABLE todos DISABLE ROW LEVEL SECURITY;

Here's what each column does:

  • id — auto-generated unique number for each todo (the database handles this)
  • text — the actual todo content you type in
  • created_at — timestamp that records when the todo was added (also automatic)

To verify it worked, click Table Editor in the left sidebar. You should see the todos table listed with 0 rows.

About Row Level Security (RLS)

We're disabling RLS for this assignment to keep things simple. In a production app, you'd use RLS policies to control who can read and write data. You'll learn about that in Module 8.

5

Install the Supabase Client Library

This package lets your React app talk to your Supabase database with simple JavaScript methods instead of raw SQL.

Make sure you're inside the react-todo-db project folder, then run:

$ npm install @supabase/supabase-js
added 1 package

You should see it added to the dependencies section of your package.json. The library provides methods like .insert(), .select(), and .delete() so you can interact with your database without writing SQL from your React components.

6

Create Environment Variables

Store your Supabase credentials in a .env.local file so they stay out of your source code and off GitHub.

In your project root (the same folder where package.json lives — not inside src), create a new file called .env.local with these two lines:

VITE_SUPABASE_URL=your_project_url_here
VITE_SUPABASE_ANON_KEY=your_anon_key_here

Replace the placeholder values

Paste your actual Project URL and anon key from Step 3. Don't leave the placeholder text.

A few important things about environment variables in Vite:

  • The VITE_ prefix is required. Vite only exposes env variables that start with VITE_ to your app. Without it, your code can't access them.
  • Restart your dev server after creating .env.local. Stop it with Ctrl+C and run npm run dev again. Environment variables only load when the server starts.
  • Check your .gitignore. Vite projects include .env.local in .gitignore by default, but verify it's there. Never commit API keys to GitHub.
7

Create the Supabase Client

Set up a small utility file that creates the connection to your Supabase project. Every component that needs database access will import from this file.

Inside src, create a lib folder, then create a file called supabaseClient.js inside it. The full path is src/lib/supabaseClient.js:

import { createClient } from '@supabase/supabase-js'

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY

export const supabase = createClient(supabaseUrl, supabaseAnonKey)

What's happening here

import.meta.env is how Vite exposes environment variables to your code. createClient establishes a connection to your Supabase project using those credentials. Any component in your app can now import { supabase } from this file and start querying the database.

9

Fetch Todos on Page Load

Use useEffect to automatically load todos from Supabase when the page opens, so users see their existing data right away.

Add this code inside your App component, right after the three useState lines:

useEffect(() => {
  fetchTodos()
}, [])

async function fetchTodos() {
  setLoading(true)
  const { data, error } = await supabase
    .from('todos')
    .select('*')
    .order('created_at', { ascending: true })

  if (error) {
    console.error('Error fetching todos:', error)
  } else {
    setTodos(data)
  }
  setLoading(false)
}

Here's what's going on:

  • useEffect(fn, []) runs once when the component first mounts — think of it as "do this when the page loads."
  • The fetchTodos function is async because database calls take time.
  • .from('todos').select('*') reads all rows from the todos table.
  • .order('created_at', { ascending: true }) sorts them oldest-first so they appear in the order you added them.
  • Supabase returns an object with data and error properties. If it worked, we put the data into state. If not, we log the error.
10

Update the Add Todo Function

Replace your Module 5 handleSubmit function with an async version that saves todos to Supabase instead of just local state.

const handleSubmit = async (e) => {
  e.preventDefault()
  if (!inputValue.trim()) return

  const { data, error } = await supabase
    .from('todos')
    .insert({ text: inputValue.trim() })
    .select()

  if (error) {
    console.error('Error adding todo:', error)
  } else {
    setTodos([...todos, data[0]])
    setInputValue('')
  }
}

Key differences from Module 5:

  • The function is now async — database calls take time, so we need await.
  • .insert() saves to Supabase, not just local state. Your todo is written to the Postgres database.
  • .select() after .insert() tells Supabase to return the newly created row, including the auto-generated id and created_at values. Without .select(), you'd get a success status but no data back.
  • No more Date.now() for IDs — the database generates unique IDs for you.
11

Update the Delete Todo Function

Replace your Module 5 deleteTodo function with an async version that removes the todo from both Supabase and local state.

const deleteTodo = async (id) => {
  const { error } = await supabase
    .from('todos')
    .delete()
    .eq('id', id)

  if (error) {
    console.error('Error deleting todo:', error)
  } else {
    setTodos(todos.filter(todo => todo.id !== id))
  }
}

How .eq() works

.eq('id', id) means "where the id column equals this value." It targets the exact row to delete in the database. After the database delete succeeds, we update local state with filter() to remove it from the UI too.

Checkpoint: Complete App.jsx

Before moving on, make sure your App.jsx matches this. If something isn't working, replace your entire file with this code:

import { useState, useEffect } from 'react'
import { supabase } from './lib/supabaseClient'
import './App.css'

function App() {
  const [todos, setTodos] = useState([])
  const [inputValue, setInputValue] = useState('')
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    fetchTodos()
  }, [])

  async function fetchTodos() {
    setLoading(true)
    const { data, error } = await supabase
      .from('todos')
      .select('*')
      .order('created_at', { ascending: true })

    if (error) {
      console.error('Error fetching todos:', error)
    } else {
      setTodos(data)
    }
    setLoading(false)
  }

  const handleSubmit = async (e) => {
    e.preventDefault()
    if (!inputValue.trim()) return

    const { data, error } = await supabase
      .from('todos')
      .insert({ text: inputValue.trim() })
      .select()

    if (error) {
      console.error('Error adding todo:', error)
    } else {
      setTodos([...todos, data[0]])
      setInputValue('')
    }
  }

  const deleteTodo = async (id) => {
    const { error } = await supabase
      .from('todos')
      .delete()
      .eq('id', id)

    if (error) {
      console.error('Error deleting todo:', error)
    } else {
      setTodos(todos.filter(todo => todo.id !== id))
    }
  }

  return (
    <div className="app">
      <h1>React Todo App</h1>

      <form className="todo-form" onSubmit={handleSubmit}>
        <input
          type="text"
          placeholder="Add a new todo..."
          value={inputValue}
          onChange={(e) => setInputValue(e.target.value)}
        />
        <button type="submit">Add</button>
      </form>

      {loading ? (
        <p>Loading todos...</p>
      ) : (
        <ul className="todo-list">
          {todos.map(todo => (
            <li key={todo.id} className="todo-item">
              <span>{todo.text}</span>
              <button
                className="delete-btn"
                onClick={() => deleteTodo(todo.id)}
              >
                Delete
              </button>
            </li>
          ))}
        </ul>
      )}
    </div>
  )
}

export default App

Test Locally

Make sure your dev server is running:

$ npm run dev

Now test the full flow:

  1. Add a todo — type something and click Add (or press Enter)
  2. Check your Supabase dashboard — go to Table Editor, click the todos table, and your new todo should be sitting right there in the database
  3. Refresh the browser page — your todos should still be there. In Module 5, they would have disappeared. That's database persistence.
  4. Delete a todo — click Delete, then check Supabase again. The row should be gone from the table too.

The moment it clicks

Refreshing the page and seeing your todos still there is the whole point. That's the difference between storing data in memory (gone on refresh) vs. storing it in a database (permanent until you delete it).

Deploy to Vercel

Push your code to GitHub and deploy to Vercel — but this time you also need to configure environment variables on Vercel's side.

Push to GitHub

If you copied your Module 5 project (Option A), you need a new GitHub repo. Create one called react-todo-db on GitHub, then update your remote and push:

$ git remote set-url origin https://github.com/yourusername/react-todo-db.git
$ git add .
$ git commit -m "Add Supabase database integration"
$ git push -u origin main

If you started fresh (Option B), initialize git and push like you did for Module 5.

Replace yourusername

Use your actual GitHub username in the remote URL.

Add environment variables in Vercel

Your .env.local file doesn't get pushed to GitHub (it's in .gitignore). So Vercel doesn't have your Supabase credentials yet. You need to add them manually:

  1. Go to your Vercel dashboard and click on your project
  2. Go to Settings ➤ Environment Variables
  3. Add VITE_SUPABASE_URL with your Project URL as the value
  4. Add VITE_SUPABASE_ANON_KEY with your anon key as the value
  5. Make sure both have the "Production" checkbox checked

You must redeploy after adding env vars

Environment variables are baked in at build time. After adding them, go to the Deployments tab, click the three dots (⋯) on the latest deployment, and select Redeploy. Without this step, your live app won't be able to connect to Supabase.

Verify Your Deployment

Make sure everything works on the live URL before submitting.

Open your Vercel URL in an incognito/private browser window (this simulates a fresh user with no cached data). Then test:

  1. Add a todo on the live site
  2. Refresh the page — the todo should still be there
  3. Delete the todo — it should disappear
  4. Check your Supabase Table Editor to confirm the data matches what you see in the browser

If your app loads but todos don't persist, your environment variables on Vercel are probably missing or incorrect. Double-check them in Settings and redeploy.

Quick debugging tip

Open the browser console (F12 ➤ Console tab) on your deployed site. If you see errors about Supabase connection failures, your env vars aren't configured correctly on Vercel. If you see CORS errors, make sure your Project URL is correct.
