4

Define a TypeScript Interface for Todos

Before adding state, define the shape of a Todo object using a TypeScript interface. This is where TypeScript really shines — it tells your editor (and you) exactly what data each todo contains.

Add this above your App function in App.tsx:

// src/App.tsx
import { useState } from 'react'
import './App.css'

// Define what a Todo looks like
interface Todo {
  id: number
  text: string
}

Now add state inside the App function, before the return. Notice the TypeScript type annotations on useState:

function App() {
  // State for the list of todos — typed as an array of Todo objects
  const [todos, setTodos] = useState<Todo[]>([])

  // State for the input field — TypeScript infers this as string
  const [inputValue, setInputValue] = useState('')

  return (
    <div className="app">
      <h1>React Todo App</h1>
    </div>
  )
}

What's Happening Here?

useState<Todo[]>([]) tells TypeScript: "this state is an array of Todo objects, starting empty." Now if you try to add an object that's missing id or text, TypeScript will show a red squiggly line before you even run the code. For inputValue, TypeScript automatically infers the type as string from the initial value '', so we don't need to write useState<string> explicitly.

Build the Form and Input

Add an input field and button to the JSX, and connect the input to state so React controls its value.

Replace the contents of your return statement with this JSX. This adds a form with a text input and an "Add" button, plus an empty list where todos will appear:

  return (
    <div className="app">
      <h1>React Todo App</h1>

      <form className="todo-form">
        <input
          type="text"
          placeholder="Add a new todo..."
          value={inputValue}
          onChange={(e) => setInputValue(e.target.value)}
        />
        <button type="submit">Add</button>
      </form>

      <ul className="todo-list">
        {/* Todos will appear here */}
      </ul>
    </div>
  )

Save and check your browser. You should see the heading, a text input, and an "Add" button. Try typing in the input — the text should appear as you type.

Controlled Component

value={inputValue} makes React the "source of truth" for what's in the input. onChange fires on every keystroke and calls setInputValue to update state, which re-renders the input with the new value. This two-way binding is called a controlled component — React controls the input, not the browser's default behavior.

TypeScript Knows the Event Type

Hover over e in VS Code — TypeScript infers it as React.ChangeEvent<HTMLInputElement> automatically. That's why e.target.value works without any type annotations. TypeScript knows it's an input element change event.

6

Add the Submit Handler

Create a function that runs when the form is submitted. It creates a new todo and adds it to the list.

Add this function inside your App component, after the two useState lines and before the return:

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault()  // Stop the browser from refreshing the page

    // Don't add empty todos
    if (!inputValue.trim()) return

    // Create a new todo object that matches our Todo interface
    const newTodo: Todo = {
      id: Date.now(),       // Unique ID using current timestamp
      text: inputValue.trim()  // Remove extra whitespace
    }

    // Add the new todo to the existing array (spread operator creates a copy)
    setTodos([...todos, newTodo])

    // Clear the input field
    setInputValue('')
  }

Now connect this function to the form. Update the opening <form> tag to include onSubmit:

      <form className="todo-form" onSubmit={handleSubmit}>

Save and test. Type something in the input and press Enter or click Add. The input should clear — but you won't see the todo yet because we haven't built the list display. That's next.

TypeScript Event Type

Notice (e: React.FormEvent) — this tells TypeScript the parameter is a form event, which is why e.preventDefault() works without errors. Without this type annotation, TypeScript would complain because it wouldn't know what e is.

Why e.preventDefault()?

When an HTML form is submitted, the browser's default behavior is to refresh the entire page. In a React app, we don't want that — we want to handle the submission with JavaScript and update the UI without a page reload. preventDefault() stops the browser's default behavior.

The Spread Operator

[...todos, newTodo] creates a new array that contains all existing todos plus the new one at the end. We never modify the original array directly — that's a core React rule. Always create new arrays/objects when updating state.

Display the Todo List

Use JavaScript's map() function to loop through the todos array and render each one as a list item.

Replace the comment {/* Todos will appear here */} inside the <ul> with this code:

      <ul className="todo-list">
        {todos.map((todo) => (
          <li key={todo.id} className="todo-item">
            <span>{todo.text}</span>
            <button className="delete-btn">
              Delete
            </button>
          </li>
        ))}
      </ul>

Save and test. Type a todo and press Enter. You should see it appear in the list below the input! Add a few more — each one stacks below the previous. The Delete button is visible but doesn't work yet.

How map() Works Here

todos.map() loops through the array and returns a new array of JSX elements — one <li> for each todo. React renders all of them inside the <ul>. The curly braces {} around todos.map() tell JSX "this is JavaScript code, evaluate it."

The key Prop is Required

Every item in a mapped list needs a unique key prop. React uses keys to efficiently track which items changed, were added, or removed. We use todo.id (the timestamp) as our key. If you forget the key, React will show a warning in the browser console.

TypeScript Autocomplete

Try typing todo. inside the map callback — VS Code will show you id and text as autocomplete options. That's your Todo interface at work. TypeScript knows the exact shape of each item in the array.

8

Add Delete Functionality

Create a function that removes a todo by its ID, then connect it to the Delete button.

Add this function in your App component, right after handleSubmit:

  const deleteTodo = (id: number) => {
    setTodos(todos.filter((todo) => todo.id !== id))
  }

Now update the Delete button in your list to call this function. Change the button to:

            <button
              className="delete-btn"
              onClick={() => deleteTodo(todo.id)}
            >
              Delete
            </button>

Save and test. Add a few todos, then click Delete on one. It should disappear. Try deleting all of them.

How filter() Works

filter() creates a new array containing only the items that pass a test. todo.id !== id returns true for every todo except the one we want to delete. So the result is a new array with that todo removed. We pass this new array to setTodos, and React re-renders the list.

Why an Arrow Function in onClick?

onClick={() => deleteTodo(todo.id)} wraps the call in an arrow function so it only runs when clicked. If you wrote onClick={deleteTodo(todo.id)} without the arrow function, it would run immediately on render — not what we want. The arrow function creates a "wrapper" that waits for the click.

TypeScript Parameter Type

Notice (id: number) in the function signature. This ensures you can only pass a number to deleteTodo. If you accidentally passed a string, TypeScript would catch it at compile time with a red underline.

Checkpoint: Complete App.tsx

Before deploying, make sure your App.tsx matches this. If your app isn't working, replace your entire file with this code:

// src/App.tsx
import { useState } from 'react'
import './App.css'

interface Todo {
  id: number
  text: string
}

function App() {
  const [todos, setTodos] = useState<Todo[]>([])
  const [inputValue, setInputValue] = useState('')

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault()
    if (!inputValue.trim()) return

    const newTodo: Todo = {
      id: Date.now(),
      text: inputValue.trim()
    }

    setTodos([...todos, newTodo])
    setInputValue('')
  }

  const deleteTodo = (id: number) => {
    setTodos(todos.filter((todo) => todo.id !== id))
  }

  return (
    <div className="app">
      <h1>React Todo App</h1>

      <form className="todo-form" onSubmit={handleSubmit}>
        <input
          type="text"
          placeholder="Add a new todo..."
          value={inputValue}
          onChange={(e) => setInputValue(e.target.value)}
        />
        <button type="submit">Add</button>
      </form>

      <ul className="todo-list">
        {todos.map((todo) => (
          <li key={todo.id} className="todo-item">
            <span>{todo.text}</span>
            <button
              className="delete-btn"
              onClick={() => deleteTodo(todo.id)}
            >
              Delete
            </button>
          </li>
        ))}
      </ul>
    </div>
  )
}

export default App
