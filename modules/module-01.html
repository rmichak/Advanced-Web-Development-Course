Assignment: Development Environment Setup

In this assignment, you will install and configure a complete modern web development environment including Node.js, npm, Express.js, React, and Next.js. By the end, you will have a working setup ready for full-stack web development projects throughout this course.

1. Overview and Prerequisites

Before we begin, ensure you have the following:

  • A computer running macOS (Ventura or later recommended) or Windows 10/11
  • Administrator access to install software
  • A stable internet connection
  • At least 10 GB of free disk space

Understanding Versions: Software versions follow a pattern like MAJOR.MINOR.PATCH (e.g., 20.11.0). The major number (first digit) is what matters most for compatibility. When this assignment says "20.x or higher," it means any version where the major number is 20 or greater.

What is LTS? LTS stands for "Long-Term Support." These versions receive security updates and bug fixes for an extended period (typically 30 months for Node.js). Always choose the LTS version for coursework and production projects.

2. Install Node.js and npm

Node.js is the foundation of our development environment. It allows you to run JavaScript outside the browser and comes bundled with npm (Node Package Manager).

For macOS Users

Option A: Using the Official Installer (Recommended for Beginners)
  1. Visit https://nodejs.org
  2. Download the LTS version (the button on the left ‚Äî it will show a version number with "LTS" next to it)
  3. Open the downloaded .pkg file
  4. Follow the installation wizard, accepting the default options
  5. When prompted, enter your administrator password
Option B: Using Homebrew (Recommended for Developers)

If you don't have Homebrew installed, first install it:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Then install Node.js:

brew install node

For Windows Users

Option A: Using the Official Installer (Recommended)
  1. Visit https://nodejs.org
  2. Download the LTS version for Windows (the button on the left)
  3. Run the downloaded .msi installer
  4. Follow the installation wizard:
    • Accept the license agreement
    • Use the default installation path
    • Important: Check the box that says "Automatically install the necessary tools"
  5. Click Install and wait for completion
Option B: Using winget (Windows Package Manager)

Open PowerShell or Command Prompt as Administrator and run:

winget install OpenJS.NodeJS.LTS

Verify Installation

Open a new terminal window and run:

node --version

You should see output like v20.11.0, v22.5.1, or similar. The exact numbers will vary ‚Äî what matters is that the major version (first number) is 20 or higher.

Also verify npm:

npm --version

You should see a version number like 10.2.0 or higher.

Success Criteria: Node.js major version is 20 or higher, and npm major version is 10 or higher.

Troubleshooting: If the commands are not recognized, try closing and reopening your terminal. On Windows, you may need to restart your computer for the PATH changes to take effect.

3. Install Git and Set Up GitHub

Git is a version control system that tracks changes to your code. GitHub is a cloud-based platform for hosting Git repositories and collaborating with others.

Create a GitHub Account

If you don't already have a GitHub account:

  1. Visit https://github.com/signup
  2. Enter your email address and create a password
  3. Choose a username (this will be visible on your profile and repositories)
  4. Complete the verification and email confirmation
  5. Choose the Free plan

Tip: Choose a professional username ‚Äî potential employers may see your GitHub profile.

Install Git

For macOS Users

Git may already be installed. Check by running:

git --version

If Git is not installed, you'll be prompted to install the Xcode Command Line Tools. Click "Install" when prompted.

Alternatively, with Homebrew:

brew install git
For Windows Users
  1. Visit https://git-scm.com/download/win
  2. The download should start automatically
  3. Run the installer with default options, but note:
    • Default editor: Select "Use Visual Studio Code as Git's default editor"
    • PATH environment: Select "Git from the command line and also from 3rd-party software"
    • Line ending conversions: Select "Checkout Windows-style, commit Unix-style line endings"

Verify Git Installation

git --version

You should see output like git version 2.43.0. The major version should be 2 and minor version 40 or higher.

Configure Git

Set up your identity. Use the same email address you used for your GitHub account.

git config --global user.name "Your Full Name"
git config --global user.email "your.email@example.com"

Verify Git Configuration

git config --global --list

Connect Git to GitHub (SSH Key Setup)

To push code to GitHub without entering your password every time:

Step 1: Generate an SSH Key
ssh-keygen -t ed25519 -C "your.email@example.com"

Press Enter to accept defaults when prompted.

Step 2: Start the SSH Agent
eval "$(ssh-agent -s)"
Step 3: Add Your SSH Key to the Agent
ssh-add ~/.ssh/id_ed25519
Step 4: Copy Your Public Key

On macOS:

pbcopy < ~/.ssh/id_ed25519.pub

On Windows (Git Bash):

cat ~/.ssh/id_ed25519.pub | clip
Step 5: Add the Key to GitHub
  1. Go to https://github.com/settings/keys
  2. Click "New SSH key"
  3. Give it a title (e.g., "My Laptop")
  4. Paste your key into the "Key" field
  5. Click "Add SSH key"
Step 6: Test the Connection
ssh -T git@github.com

If successful, you'll see: Hi username! You've successfully authenticated...

Success Criteria: Git version is 2.40.x or higher, your name and email are configured, and you can authenticate with GitHub via SSH.

6. Create and Test an Express.js Project

Express.js is a minimal web framework for Node.js. Let's create a simple server to verify your setup.

Initialize a New Project

In your terminal (inside your WebDev folder):

mkdir express-test
cd express-test
npm init -y

Install Express.js

npm install express

Verify Express Version

npm list express

The major version should be 5 or higher. If you see Express 4.x, install version 5: npm install express@5

Create the Server File

Create a new file called server.js:

const express = require('express');
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
  res.json({
    message: 'Express server is running!',
    nodeVersion: process.version,
    expressVersion: require('express/package.json').version,
    timestamp: new Date().toISOString()
  });
});

app.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}`);
});

Run the Server

node server.js

Test the Server

  1. Open your browser to http://localhost:3000
  2. You should see a JSON response with your Node.js and Express versions
  3. Take a screenshot of this output
  4. Press Ctrl+C to stop the server

Success Criteria: The JSON response shows Express version 5.x or higher.

9. Create a Verification Script

Create a script that displays all your installed versions in one place.

Navigate to WebDev Folder

cd ..

Create the Verification Script

Create a file named verify-setup.js in your WebDev folder:

const { execSync } = require('child_process');
const path = require('path');
const fs = require('fs');

console.log('='.repeat(60));
console.log('  DEVELOPMENT ENVIRONMENT VERIFICATION');
console.log('  Generated:', new Date().toLocaleString());
console.log('='.repeat(60));
console.log();

const requirements = {
  git: 2.40, node: 20, npm: 10, express: 5, react: 19, next: 15
};

let allPassed = true;

function checkMajorVersion(name, current, minimum) {
  const majorVersion = parseInt(current.replace('v', '').split('.')[0]);
  const passed = majorVersion >= minimum;
  if (!passed) allPassed = false;
  const status = passed ? 'PASS' : 'FAIL';
  console.log(`${status} - Requires ${minimum}.x or higher, found ${current}`);
  return passed;
}

// Git
console.log('GIT');
console.log('-'.repeat(40));
try {
  const gitVersion = execSync('git --version').toString().trim();
  console.log('Version:', gitVersion);
  const gitUser = execSync('git config --global user.name').toString().trim();
  const gitEmail = execSync('git config --global user.email').toString().trim();
  console.log('User:', gitUser);
  console.log('Email:', gitEmail);
} catch (e) { console.log('FAIL - Git not installed'); allPassed = false; }
console.log();

// Node.js
console.log('NODE.JS');
console.log('-'.repeat(40));
console.log('Version:', process.version);
checkMajorVersion('Node.js', process.version, requirements.node);
console.log();

// npm
console.log('NPM');
console.log('-'.repeat(40));
const npmVersion = execSync('npm --version').toString().trim();
console.log('Version:', npmVersion);
checkMajorVersion('npm', npmVersion, requirements.npm);
console.log();

// Express
console.log('EXPRESS.JS');
console.log('-'.repeat(40));
try {
  const expressPath = path.join(__dirname, 'express-test', 'node_modules', 'express', 'package.json');
  if (fs.existsSync(expressPath)) {
    const expressPkg = require(expressPath);
    console.log('Version:', expressPkg.version);
    checkMajorVersion('Express', expressPkg.version, requirements.express);
  } else { console.log('FAIL - Not found in express-test folder'); allPassed = false; }
} catch (e) { console.log('FAIL - Not installed'); allPassed = false; }
console.log();

// React
console.log('REACT');
console.log('-'.repeat(40));
try {
  const reactPath = path.join(__dirname, 'react-test', 'node_modules', 'react', 'package.json');
  if (fs.existsSync(reactPath)) {
    const reactPkg = require(reactPath);
    console.log('Version:', reactPkg.version);
    checkMajorVersion('React', reactPkg.version, requirements.react);
  } else { console.log('FAIL - Not found in react-test folder'); allPassed = false; }
} catch (e) { console.log('FAIL - Not installed'); allPassed = false; }
console.log();

// Next.js
console.log('NEXT.JS');
console.log('-'.repeat(40));
try {
  const nextPath = path.join(__dirname, 'nextjs-test', 'node_modules', 'next', 'package.json');
  if (fs.existsSync(nextPath)) {
    const nextPkg = require(nextPath);
    console.log('Version:', nextPkg.version);
    checkMajorVersion('Next.js', nextPkg.version, requirements.next);
  } else { console.log('FAIL - Not found in nextjs-test folder'); allPassed = false; }
} catch (e) { console.log('FAIL - Not installed'); allPassed = false; }
console.log();

console.log('='.repeat(60));
if (allPassed) {
  console.log('  ALL CHECKS PASSED - Your environment is ready!');
} else {
  console.log('  SOME CHECKS FAILED - Please review the issues above');
}
console.log('='.repeat(60));

Run the Verification Script

Take a screenshot of this output.

10. Troubleshooting Common Issues

"command not found: node" or "node is not recognized"

  • Close and reopen your terminal
  • On Windows, restart your computer
  • Verify Node.js is in your PATH environment variable

SSH key not working with GitHub

  • Make sure you copied the public key (id_ed25519.pub), not the private key
  • Verify the SSH agent is running: eval "$(ssh-agent -s)"
  • Re-add your key: ssh-add ~/.ssh/id_ed25519

npm permission errors on macOS

mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

Port already in use

  • Find and stop the process using that port
  • Or use a different port by modifying the PORT variable

npm packages fail to install

  • Clear npm cache: npm cache clean --force
  • Delete node_modules and package-lock.json, then run npm install again
