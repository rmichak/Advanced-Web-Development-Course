Visual Studio Code

Your Code Editor

What is Visual Studio Code?

VS Code is a free, open-source code editor developed by Microsoft.

Key Features

  • IntelliSense: Smart code completion
  • Integrated Terminal: Run commands without leaving the editor
  • Git Integration: Built-in version control
  • Extensions: Customize with thousands of add-ons
  • Cross-Platform: Works on Windows, macOS, and Linux

Installing VS Code

  1. Visit code.visualstudio.com
  2. Click the download button for your operating system
  3. Run the installer and follow the prompts
  4. Launch VS Code after installation

Tip: During installation on Windows, check "Add to PATH" to enable opening VS Code from the command line with code .

Essential VS Code Extensions

Install these extensions for web development:

Extension Purpose
Live Server Launch a local development server with live reload
Prettier Automatic code formatting
ESLint JavaScript linting and error detection
GitLens Enhanced Git capabilities
Auto Rename Tag Automatically rename paired HTML tags

How to Install Extensions

  1. Open VS Code
  2. Click the Extensions icon in the sidebar (or press Ctrl+Shift+X)
  3. Search for the extension name
  4. Click Install

Try it now: Install the "Live Server" extension. You'll use this to preview your web pages throughout the course.

Recommended VS Code Settings

Open Settings with Ctrl+, (Windows/Linux) or Cmd+, (Mac)

Recommended Settings

  • Auto Save: Set to "afterDelay" to save automatically
  • Format On Save: Enable to auto-format code
  • Tab Size: Set to 2 for web development
  • Word Wrap: Enable for easier reading

Node.js

JavaScript Runtime Environment

What is Node.js?

Node.js lets you run JavaScript outside the browser.

Why We Need It

  • Run JavaScript on your computer
  • Use npm (Node Package Manager)
  • Build servers and APIs
  • Run development tools

What npm Does

  • Install libraries and tools
  • Manage project dependencies
  • Run scripts and commands
  • Share code with others

Installing Node.js

  1. Visit nodejs.org
  2. Download the LTS (Long Term Support) version
  3. Run the installer and accept the defaults
  4. Restart your terminal/command prompt

Important: Always download the LTS version for stability. The "Current" version may have untested features.

Verify Your Installation

Open a terminal and run these commands:

node --version
npm --version

You should see version numbers like:

v22.12.0
10.9.0

Tip: In VS Code, open the integrated terminal with Ctrl+` (backtick key).

Git

Version Control System

What is Git?

Git is a version control system that tracks changes to your code.

Why Use Git?

  • History: See every change ever made to your code
  • Backup: Never lose your work
  • Branching: Work on features without affecting main code
  • Collaboration: Work with others on the same project
  • Undo: Revert to previous versions if something breaks

Installing Git

Windows

  1. Visit git-scm.com
  2. Download the Windows installer
  3. Run installer, accept defaults
  4. Select "Use Git from Windows Command Prompt"

macOS

  1. Open Terminal
  2. Type git --version
  3. If not installed, follow the prompt to install Xcode Command Line Tools

Configure Git

Set your identity for Git commits:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Verify your configuration:

git config --list

Important: Use the same email you'll use for GitHub. This links your commits to your GitHub account.

GitHub

Cloud-Based Code Hosting

Git vs GitHub

Git

  • Software on your computer
  • Tracks changes locally
  • Works offline
  • Free and open source

GitHub

  • Website/cloud service
  • Stores your code online
  • Enables collaboration
  • Free for public repositories

Think of it this way: Git is the tool, GitHub is where you store and share your work.

Create a GitHub Account

  1. Visit github.com
  2. Click Sign Up
  3. Enter your email, create a password, choose a username
  4. Verify your email address
  5. Complete the welcome survey (optional)

Username Tip: Choose a professional username. Employers may see your GitHub profile!

Set Up SSH Keys

SSH keys let you connect to GitHub without entering your password each time.

Generate a new SSH key:

ssh-keygen -t ed25519 -C "your.email@example.com"

Press Enter to accept the default file location, then set a passphrase (optional).

Add SSH Key to GitHub

  1. Copy your public key: 
    cat ~/.ssh/id_ed25519.pub
  2. Go to GitHub.com → Settings → SSH and GPG Keys
  3. Click New SSH Key
  4. Paste your key and give it a title (e.g., "My Laptop")
  5. Click Add SSH Key

Test Your Connection

Verify SSH is working:

ssh -T git@github.com

You should see:

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

Success! You can now push and pull code to GitHub without entering your password.

Basic Git Workflow

Essential Commands

The Git Workflow

Four Steps to Save Your Work

  1. Make changes to your files
  2. Stage the changes (git add)
  3. Commit with a message (git commit)
  4. Push to GitHub (git push)

Clone a Repository

Download a repository from GitHub to your computer:

git clone git@github.com:username/repository-name.git

This creates a folder with all the project files and Git history.

Tip: Use the SSH URL (starts with git@github.com) when you've set up SSH keys.

Check Repository Status

See what files have changed:

git status

This shows:

  • Untracked files: New files Git doesn't know about
  • Modified files: Changed files not yet staged
  • Staged files: Files ready to be committed

Stage Your Changes

Add files to the staging area:

# Add a specific file
git add filename.html

# Add all changed files
git add .

# Add all files of a type
git add *.css

Staging means selecting which changes to include in your next commit.

Commit Your Changes

Save your staged changes with a message:

git commit -m "Add navigation menu to homepage"

Good Commit Messages

  • Start with a verb: "Add", "Fix", "Update", "Remove"
  • Be specific about what changed
  • Keep it under 50 characters

Push to GitHub

Upload your commits to GitHub:

git push origin main

This sends your local commits to the remote repository on GitHub.

origin = the remote repository (GitHub)

main = the branch name

Pull from GitHub

Download changes from GitHub to your computer:

git pull origin main

Always pull before starting work to get the latest changes!

Best Practice: Pull at the start of each coding session to avoid conflicts.

Git Commands Summary

Command Purpose
git clone [url] Download a repository
git status Check what's changed
git add . Stage all changes
git commit -m "msg" Save changes locally
git push Upload to GitHub
git pull Download from GitHub

Tips for Success

Do

  • Commit often with clear messages
  • Pull before you start working
  • Push your work daily
  • Read error messages carefully

Don't

  • Commit sensitive data (passwords, API keys)
  • Make huge commits with many changes
  • Ignore merge conflicts
  • Be afraid to ask for help
