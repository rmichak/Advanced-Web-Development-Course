React Fundamentals

What is React?

A JavaScript library for building user interfaces, focused on components.

Library, Not Framework

Focuses on UI layer only. You choose other tools for routing, etc.

Component-Based

Build UIs from small, reusable pieces that manage their own state.

Declarative

Describe what UI should look like, React handles the updates.

Created by Meta (Facebook) in 2013

Why React?

React solves real problems in building modern web applications.

Declarative UI

Describe the desired state, React handles DOM updates automatically

Virtual DOM

Efficient updates - only changes what's necessary, not the whole page

Massive Ecosystem

Thousands of libraries, tools, and learning resources available

Your First Component

Components are functions that return JSX - React's way of describing UI.

// Greeting.jsx
function Greeting() {
  return (
    <div>
      <h1>Hello, React!</h1>
      <p>Welcome to my first component.</p>
    </div>
  );
}

export default Greeting;

Capital Letter Required

Component names must start with uppercase: Greeting, not greeting

Return One Element

Wrap multiple elements in a single parent div or fragment

Understanding JSX

JSX lets you write HTML-like syntax in JavaScript - with a few differences.

HTML

<div class="container">
  <label for="name">Name</label>
  <img src="pic.jpg">
</div>

JSX

<div className="container">
  <label htmlFor="name">Name</label>
  <img src="pic.jpg" />
</div>

Key Differences

class ‚Üí className | for ‚Üí htmlFor | All tags must close: <img />

JSX Expressions

Use curly braces to embed any JavaScript expression in your JSX.

function Profile() {

Part 2

State & Interactivity

Understanding Props

Props pass data from parent to child components - like function arguments.

// Parent passes props
function App() {
  return <UserCard name="Alice" age={25} />;
}

// Child receives props
function UserCard(props) {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>Age: {props.age}</p>
    </div>
  );
}

Props are Read-Only

Never modify props directly. Data flows one way: parent ‚Üí child.

Props with Destructuring

Destructure props for cleaner code and add default values.

// Destructure in parameters - cleaner!
function UserCard({ name, age, role = 'User' }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Age: {age}</p>
      <p>Role: {role}</p>
    </div>
  );
}

// Usage - role will default to 'User'
<UserCard name="Alice" age={25} />
<UserCard name="Bob" age={30} role="Admin" />

Introduction to State

State is data that changes over time - when it updates, the component re-renders.

Props State
Passed from parent Internal to component
Read-only (immutable) Can change (mutable)
Like function arguments Like component memory

Examples of state: form inputs, counters, toggle switches, fetched data

The useState Hook

useState adds state to your components - returns the value and a setter function.

import { useState } from 'react';

function Counter() {
  // Declare state: [currentValue, setterFunction]
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}

Never Modify State Directly

count = 5 ‚ùå Wrong | setCount(5) ‚úì Correct

Handling Events

React events use camelCase and pass functions, not strings.

function ButtonDemo() {
  const handleClick = () => {
    alert('Button clicked!');
  };

  return (
    <div>
      {/* Named function */}
      <button onClick={handleClick}>Click Me</button>

      {/* Inline arrow function */}
      <button onClick={() => alert('Hi!')}>Say Hi</button>
    </div>
  );
}

onClick

Clicks, taps

onChange

Input changes

onSubmit

Form submission

Rendering Lists

Use map() to transform arrays into JSX elements - always include a unique key.

function FruitList() {
  const fruits = [
    { id: 1, name: 'Apple' },
    { id: 2, name: 'Banana' },
    { id: 3, name: 'Cherry' }
  ];

  return (
    <ul>
      {fruits.map(fruit => (
        <li key={fruit.id}>{fruit.name}</li>
      ))}
    </ul>
  );
}

Why Keys Matter

Keys help React track items efficiently. Use unique IDs, not array indices.

4Add State for Todos

Use useState to track the list of todos and the input field value.

function App() {
  // State for the list of todos
  const [todos, setTodos] = useState([])

  // State for the input field
  const [inputValue, setInputValue] = useState('')

  return (
    // ... rest of JSX
  )
}

Two Pieces of State

todos - array of todo objects | inputValue - current text in input

5Build the Input Section

Make the input a controlled component by connecting it to state.

<form className="todo-form">
  <input
    type="text"
    placeholder="Add a new todo..."
    value={inputValue}
    onChange={(e) => setInputValue(e.target.value)}
  />
  <button type="submit">Add</button>
</form>

Controlled Component

value={inputValue} displays state | onChange updates state when typing

6Add Todo Function

Create a function to handle form submission and add new todos.

const handleSubmit = (e) => {
  e.preventDefault()  // Prevent page refresh

  if (!inputValue.trim()) return  // Don't add empty todos

  // Create new todo object
  const newTodo = {
    id: Date.now(),  // Unique ID
    text: inputValue.trim()
  }

  // Add to list and clear input
  setTodos([...todos, newTodo])
  setInputValue('')
}

// Add to form:
<form onSubmit={handleSubmit}>

7Display the Todo List

Use map() to render each todo as a list item with a delete button.

8Delete Todos

Add a function to remove todos using filter().

// Add this function in your App component
const deleteTodo = (id) => {
  setTodos(todos.filter(todo => todo.id !== id))
}

// Update the delete button:
<button
  className="delete-btn"
  onClick={() => deleteTodo(todo.id)}
>
  Delete
</button>

How filter() Works

Creates new array with items where todo.id !== id is true (keeps non-matching)
